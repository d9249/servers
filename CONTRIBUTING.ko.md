## 서버 구현

환영하는 사항:
- **버그 수정** — 성가신 버그들을 제거하는데 도움을 주세요.
- **사용성 개선** — 사람과 에이전트가 서버를 더 쉽게 사용할 수 있도록 만들어 주세요.
- **MCP 프로토콜 기능을 보여주는 개선사항** — Tools뿐만 아니라 Resources, Prompts 또는 Roots와 같은 MCP 프로토콜의 덜 활용되는 측면을 참조 서버가 더 잘 보여줄 수 있도록 돕는 기여를 권장합니다. 예를 들어, filesystem-server에 Roots 지원을 추가하면 이 중요하지만 잘 알려지지 않은 기능을 보여주는데 도움이 됩니다.

좀 더 신중하게 검토하는 사항:
- **기타 새로운 기능** — 특히 서버의 핵심 목적에 중요하지 않거나 매우 주관적인 경우입니다. 기존 서버들은 커뮤니티에 영감을 주기 위한 참조 서버입니다. 특정 기능이 필요하다면 향상된 버전을 직접 만드는 것을 권장합니다! 우리는 다양한 서버 생태계가 모두에게 유익하다고 생각하며, README에 여러분의 개선된 서버를 링크하는 것을 환영합니다.

받지 않는 사항:
- **새로운 서버 구현** — 직접 게시하고 README에서 링크하는 것을 권장합니다.

## 테스트

TypeScript로 구현된 서버의 테스트를 추가하거나 구성할 때는 **vitest**를 테스트 프레임워크로 사용하세요. Vitest는 더 나은 ESM 지원, 더 빠른 테스트 실행 및 더 현대적인 테스트 경험을 제공합니다.

## 문서화

기존 문서의 개선을 환영합니다 - 가능하다면 문제점을 문서화하는 것보다 인체공학적 개선을 선호합니다!

완전히 새로운 문서를 추가하는 것에 대해서는 더 신중하게 검토하며, 특히 벤더 중립적이지 않은 방식(예: 특정 클라이언트로 특정 서버를 실행하는 방법)은 더욱 그렇습니다.
